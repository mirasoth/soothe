## Issue #1: Tool Outputs Leak After Completion

### Problem

**Observation**:
- Completion message at line 15: `● analyze this project arch (complete, 4 steps) (80.3s)`
- Tool outputs continue from lines 16-1168 (1153 lines!)
- File listings, README content, pyproject.toml, etc. all appear AFTER completion

**Root Cause**: Line 353 in `renderer.py` clears `agentic_stdout_suppressed = False` immediately on `loop.completed`, allowing late tool results to leak through.

```python
# BUGGY CODE (line 353):
if event_type == "soothe.agentic.loop.completed":
    ...
    self._state.agentic_stdout_suppressed = False  # ⚠️ TOO EARLY!
```

**Why This Happens**: Tool execution continues in background after `loop.completed` event. The daemon sends tool results after the completion event, but suppression was already cleared.

### Fix Applied

**File**: `src/soothe/ux/cli/renderer.py` (line 353)

```python
# FIXED CODE:
if event_type == "soothe.agentic.loop.completed":
    ...
    # Note: Keep agentic_stdout_suppressed=True to prevent late tool result leaks
    # Will be cleared on status=idle or new thread session
```

**Impact**: `agentic_stdout_suppressed` stays `True` until status changes to idle or new thread starts, blocking all late tool outputs.

### Verification Test

```python

1. Loop started → agentic_stdout_suppressed = True ✅
2. Loop completed → agentic_stdout_suppressed = True ✅ (FIXED!)
3. Late tool result → suppressed ✅
```

## Issue #2: Progress NOT Progressive

### Problem

**Expected**: Each iteration shows actions "closer towards the goal"

**Actual**: Progress regresses from specific back to generic

```
Iteration 1: "Use file and shell tools in the workspace to gather facts..."
           → Generic, not goal-specific

Iteration 2: "Use file and shell tools in the workspace to gather facts..."
           → REPEATED - NO PROGRESS ❌

Iteration 3: "List the root directory structure of the Soothe project to understand overall organization"
           → Specific, concrete action ✅ GOOD!

Iteration 4: "Use file and shell tools in the workspace to gather facts..."
           → REGRESSED back to generic ❌ BAD!
```

### Analysis

**Problem**: Action descriptions generated by LLM are:
1. Often generic ("Use file and shell tools...")
2. Repeated across iterations
3. Don't show progressive refinement
4. Regress from specific back to generic

**Root Cause**: `StreamDisplayPipeline._on_loop_agent_reason()` extracts actions from:
1. `user_summary` field
2. `soothe_next_action` field
3. Status fallback

But these fields are generated by the LLM's reasoning, which doesn't maintain progressive improvement.

### Impact on IG-143 Success Criteria

**New Criterion Needed**:
> "Each iteration's action description should show measurable progress toward the goal (specificity increases, not decreases)"

**Current Status**: ❌ FAIL - Progress regresses

**Fix Required**: Improve action extraction logic OR guide LLM to generate better progressive actions
