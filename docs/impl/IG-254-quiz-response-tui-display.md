# IG-254: Fix Quiz Response Display in TUI/CLI

## Status
✅ Completed

## Summary
Fix quiz response not being displayed in TUI/CLI by adding quiz response event handling to event processor.

## Problem Statement
Quiz responses were successfully generated by the daemon (events were logged) but never displayed to users in TUI/CLI:

```
2026-04-24 19:52:45,327 INFO     [uopv1vvofxfd] soothe.core.runner._runner_agentic:373 [Intent] Quiz → direct LLM response
2026-04-24 19:52:45,328 DEBUG    [uopv1vvofxfd] soothe.core.runner._runner_phases:168 [IG-250] Quiz completed (piggybacked) for query: capital of france
```

User saw: **No response returned to TUI/CLI**

## Root Cause Analysis
Quiz intent classification (IG-250) added quiz events and handling in daemon, but TUI/CLI event processor (`event_processor.py`) did not handle `soothe.output.quiz.responded` events as assistant text.

The processor only handled:
- `soothe.output.chitchat.responded` → Chitchat responses
- `soothe.output.autonomous.final_report.reported.reported` → Final reports

Quiz responses (`soothe.output.quiz.responded`) were ignored.

## Implementation

### Fix Location
**File**: `packages/soothe-cli/src/soothe_cli/shared/event_processor.py`

### Change
Add `"soothe.output.quiz.responded"` to event types processed as assistant text:

```python
# Handle chitchat/quiz/final responses through shared cleaner path
if etype in {
    "soothe.output.chitchat.responded",
    "soothe.output.quiz.responded",  # ← Added
    "soothe.output.autonomous.final_report.reported.reported",
}:
    content = data.get("content", data.get("summary", ""))
    if content and self._presentation.tier_visible(VerbosityTier.QUIET, self._verbosity):
        cleaned = self._clean_assistant_text(content)
        if cleaned:
            self._emit_assistant_text(
                cleaned,
                is_main=True,
                is_streaming=False,
            )
            self._presentation.mark_final_answer_locked()
    return
```

### How It Works
1. Quiz intent classified → `QuizResponseEvent` emitted with content
2. Event processor receives `soothe.output.quiz.responded` event
3. Event now matches handler → processed as assistant text
4. Response displayed to user in TUI/CLI

### Event Types Comparison
| Event Type | Handler | Display |
|------------|---------|---------|
| `soothe.output.chitchat.responded` | ✅ Assistant text | ✅ Displayed |
| `soothe.output.quiz.responded` | ✅ Assistant text (after fix) | ✅ Displayed |
| `soothe.output.autonomous.final_report.reported.reported` | ✅ Assistant text | ✅ Displayed |
