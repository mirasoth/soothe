# REST API Specification Reference

**Status**: Reference Document
**Source**: Extracted from RFC-0013
**Purpose**: HTTP REST API specification for Soothe daemon

## Overview

The HTTP REST API provides CRUD operations for thread management, configuration, file operations, and system monitoring. It complements the WebSocket protocol for real-time streaming with standard HTTP patterns suitable for web applications and mobile clients.

**Design Principles**:
- RESTful resource naming conventions
- JSON request/response bodies
- Async FastAPI implementation for performance
- OpenAPI documentation auto-generated
- Shared authentication with WebSocket
- CORS support for web clients

**Base URL**: `http://localhost:8766/api/v1`

**Transport Configuration**:
- `enabled`: Enable/disable HTTP REST server
- `host`: Bind address (default: `127.0.0.1`)
- `port`: Listen port (default: `8766`)
- `tls_enabled`: Enable TLS encryption
- `tls_cert`, `tls_key`: TLS certificate paths
- `cors_origins`: Allowed CORS origins

## Endpoints

### Thread Management

#### List Threads

```http
GET /api/v1/threads
```

**Description**: List all threads with optional filtering.

**Query Parameters**:
- `status` (optional): Filter by thread status (`active`, `archived`, `suspended`)
- `limit` (optional): Maximum number of threads to return (default: 50)
- `offset` (optional): Pagination offset (default: 0)

**Response**: `200 OK`
```json
{
  "threads": [
    {
      "thread_id": "abc123",
      "status": "active",
      "created_at": "2026-03-19T10:00:00Z",
      "updated_at": "2026-03-19T12:30:00Z",
      "message_count": 15,
      "last_query": "Analyze authentication code"
    }
  ],
  "total": 1,
  "limit": 50,
  "offset": 0
}
```

#### Get Thread

```http
GET /api/v1/threads/{thread_id}
```

**Description**: Get detailed information about a specific thread.

**Path Parameters**:
- `thread_id` (required): Thread identifier

**Response**: `200 OK`
```json
{
  "thread_id": "abc123",
  "status": "active",
  "created_at": "2026-03-19T10:00:00Z",
  "updated_at": "2026-03-19T12:30:00Z",
  "message_count": 15,
  "last_query": "Analyze authentication code",
  "artifacts": ["report.md", "analysis.json"],
  "metadata": {
    "mode": "interactive",
    "autonomous": false
  }
}
```

**Error Response**: `404 Not Found`
```json
{
  "error": "Thread not found",
  "thread_id": "abc123"
}
```

#### Create Thread

```http
POST /api/v1/threads
```

**Description**: Create a new thread.

**Request Body**:
```json
{
  "thread_id": "custom-id" // optional, auto-generated if omitted
}
```

**Response**: `201 Created`
```json
{
  "thread_id": "xyz789",
  "status": "active",
  "created_at": "2026-03-19T14:00:00Z"
}
```

#### Archive Thread

```http
DELETE /api/v1/threads/{thread_id}
```

**Description**: Archive a thread (soft delete).

**Path Parameters**:
- `thread_id` (required): Thread identifier

**Response**: `204 No Content`

#### Resume Thread

```http
POST /api/v1/threads/{thread_id}/resume
```

**Description**: Resume an archived or suspended thread.

**Path Parameters**:
- `thread_id` (required): Thread identifier

**Response**: `200 OK`
```json
{
  "thread_id": "abc123",
  "status": "active",
  "resumed_at": "2026-03-19T15:00:00Z"
}
```

#### Get Thread Messages

```http
GET /api/v1/threads/{thread_id}/messages
```

**Description**: Get message history for a thread (event log).

**Path Parameters**:
- `thread_id` (required): Thread identifier

**Query Parameters**:
- `limit` (optional): Maximum messages to return (default: 100)
- `offset` (optional): Pagination offset (default: 0)
- `kind` (optional): Filter by message kind (`event`, `tool_call`, `tool_result`, `conversation`)

**Response**: `200 OK`
```json
{
  "messages": [
    {
      "timestamp": "2026-03-19T10:05:00Z",
      "kind": "conversation",
      "role": "user",
      "content": "Analyze authentication code"
    },
    {
      "timestamp": "2026-03-19T10:05:02Z",
      "kind": "event",
      "type": "soothe.cognition.plan.created",
      "data": {"goal": "Analyze auth", "steps": [...]}
    }
  ],
  "total": 150,
  "limit": 100,
  "offset": 0
}
```

#### Get Thread Artifacts

```http
GET /api/v1/threads/{thread_id}/artifacts
```

**Description**: List artifacts generated by a thread.

**Path Parameters**:
- `thread_id` (required): Thread identifier

**Response**: `200 OK`
```json
{
  "artifacts": [
    {
      "filename": "report.md",
      "size_bytes": 4096,
      "created_at": "2026-03-19T10:10:00Z",
      "download_url": "/api/v1/files/report.md"
    }
  ]
}
```

### Configuration

#### Get Configuration

```http
GET /api/v1/config
```

**Description**: Get current daemon configuration.

**Response**: `200 OK`
```json
{
  "model": "claude-sonnet-4-6",
  "protocols": {
    "context": {"max_tokens": 8000},
    "memory": {"enabled": true},
    "planner": {"enabled": true}
  },
  "transports": {
    "unix_socket": {"enabled": true},
    "websocket": {"enabled": false, "port": 8765},
    "http_rest": {"enabled": false, "port": 8766}
  }
}
```

#### Update Configuration

```http
PUT /api/v1/config
```

**Description**: Update daemon configuration (requires restart for some settings).

**Request Body**:
```json
{
  "model": "claude-opus-4-6",
  "protocols": {
    "memory": {"enabled": false}
  }
}
```

**Response**: `200 OK`
```json
{
  "message": "Configuration updated",
  "restart_required": true,
  "affected_keys": ["model"]
}
```

#### Get Configuration Schema

```http
GET /api/v1/config/schema
```

**Description**: Get JSON schema for configuration validation.

**Response**: `200 OK`
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "model": {"type": "string"},
    "protocols": {"type": "object"}
  }
}
```

### File Operations

#### Upload File

```http
POST /api/v1/files/upload
```

**Description**: Upload a file (multipart form data).

**Request**: `multipart/form-data`
- `file`: File content
- `thread_id` (optional): Associate with thread

**Response**: `201 Created`
```json
{
  "file_id": "file_abc123",
  "filename": "document.pdf",
  "size_bytes": 102400,
  "content_type": "application/pdf",
  "uploaded_at": "2026-03-19T16:00:00Z"
}
```

#### Download File

```http
GET /api/v1/files/{file_id}
```

**Description**: Download a file by ID.

**Path Parameters**:
- `file_id` (required): File identifier

**Response**: `200 OK`
- Content-Type: `<mime-type>`
- Content-Disposition: `attachment; filename="<filename>"`
- Body: File content (binary)

#### Delete File

```http
DELETE /api/v1/files/{file_id}
```

**Description**: Delete a file.

**Path Parameters**:
- `file_id` (required): File identifier

**Response**: `204 No Content`

#### Shutdown Daemon

```http
POST /api/v1/shutdown
```

**Description**: Gracefully shutdown the daemon.

**Request Body**:
```json
{
  "force": false // optional, force immediate shutdown
}
```

**Response**: `202 Accepted`
```json
{
  "message": "Shutdown initiated",
  "grace_period_seconds": 5
}
```

## Error Responses

All endpoints follow standard HTTP error format:

```json
{
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": {} // optional, additional context
}
```

**Standard HTTP Status Codes**:
- `400 Bad Request` - Invalid request body or parameters
- `401 Unauthorized` - Authentication required
- `403 Forbidden` - Insufficient permissions
- `404 Not Found` - Resource not found
- `409 Conflict` - Resource conflict (e.g., thread already exists)
- `429 Too Many Requests` - Rate limit exceeded
- `500 Internal Server Error` - Server-side error
- `503 Service Unavailable` - Daemon not available
