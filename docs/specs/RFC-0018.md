# RFC-0018: Subagent Extension Specification

**RFC**: 0018
**Title**: Subagent Extension Specification
**Status**: Draft
**Kind**: Architecture Design
**Created**: 2026-03-23
**Dependencies**: RFC-0001, RFC-0002, RFC-0008, RFC-0013
**Related**: RFC-0015

## Abstract

This RFC defines a standardized extension specification for Soothe subagents, enabling third-party developers to create, distribute, and integrate custom subagents with clear lifecycle management, dependency declaration, and security boundaries. The specification defines a `SubagentModule` protocol, manifest schema, discovery mechanisms, and configuration integration that preserve Soothe's protocol-first architecture while enabling a vibrant extension ecosystem.

## Motivation

### Problem Statement

Current subagent integration follows an ad-hoc pattern with several limitations:

1. **No standard entry point**: Each subagent defines its own factory signature (`create_<name>_subagent`) with varying parameter names and types
2. **Manual registration**: Factories must be added to `SUBAGENT_FACTORIES` dictionary in `core/resolver/_resolver_tools.py`, requiring code changes
3. **No dependency declaration**: Subagents cannot declare required protocols, tools, or external libraries, leading to runtime failures
4. **No lifecycle hooks**: No standardized `initialize()`, `shutdown()`, or `health_check()` methods for resource management
5. **Limited discovery**: Only filesystem-based discovery via Weaver's `GeneratedAgentRegistry` for dynamically generated agents
6. **No configuration schema**: Subagent-specific configuration is untyped `dict[str, Any]` with no validation
7. **No security model**: All subagents run with full orchestrator permissions, no trust levels or permission boundaries

### Design Goals

1. **Standardized entry point**: `SubagentModule` protocol with clear lifecycle methods and factory signature
2. **Declarative manifests**: Rich metadata, dependencies, capabilities, and configuration schemas
3. **Multiple discovery mechanisms**: Python entry points, config declarations, filesystem scanning
4. **Dependency resolution**: Automatic checking of protocol, tool, library, and configuration requirements
5. **Configuration integration**: Subagent-specific JSON schemas that extend `SootheConfig`
6. **Security model**: Trust levels (built-in, trusted, standard, untrusted), permissions, and access control
7. **Backward compatibility**: Existing subagents (browser, claude, skillify, weaver) work unchanged
8. **Graceful degradation**: Missing dependencies or failed initialization disables the subagent, not the orchestrator

### Non-Goals

- **Code signing and verification**: Future enhancement for cryptographically signed extensions
- **Extension marketplace**: Central registry for publishing and discovering extensions
- **Hot reloading**: Dynamic loading/unloading without orchestrator restart
- **Sandboxing**: Process isolation or WASM-based sandboxing for untrusted extensions

## Guiding Principles

### Principle 1: Protocol-First Design

Subagent extensions are defined as protocols (`typing.Protocol`), not concrete implementations. The `SubagentModule` protocol specifies the interface; implementations are swappable via configuration. This maintains Soothe's architectural principle of runtime-agnostic protocols.

### Principle 2: Explicit Over Implicit

Subagents explicitly declare dependencies, capabilities, and configuration schemas in their manifest. No runtime introspection, magic discovery, or implicit dependencies. Everything needed to validate and load a subagent is declared upfront.

### Principle 3: Graceful Degradation

Missing dependencies, failed imports, and configuration errors result in disabled subagents, not orchestrator startup failures. The system continues with available capabilities. Required vs. optional dependencies determine whether a subagent can load.

### Principle 4: Security by Default

Unsigned third-party subagents run with restricted permissions. Built-in subagents are trusted by default. Explicit user approval required for elevated permissions. Permission profiles enforce least-privilege access.

### Principle 5: Deepagents Compatibility

All extension subagents return deepagents `SubAgent` or `CompiledSubAgent` types. No new runtime types. Extensions integrate seamlessly with the existing `task` tool and subagent middleware.

## Specification

### 1. SubagentModule Protocol

The core protocol that all extension modules MUST implement.

```python
from typing import Protocol, runtime_checkable, Any
from pydantic import BaseModel

@runtime_checkable
class SubagentModule(Protocol):
    """Protocol for subagent extension modules.

    All subagent extensions MUST implement this protocol.
    Lifecycle hooks are optional (provide no-op defaults).

    Example:
        class MySubagentModule:
            @property
            def manifest(self) -> SubagentManifest:
                return SubagentManifest(
                    name="my_subagent",
                    version="1.0.0",
                    description="My custom subagent",
                )

            async def initialize(self, config: SootheConfig) -> None:
                # Set up resources, validate config
                pass

            def create_subagent(
                self,
                model: str | BaseChatModel | None = None,
                *,
                config: SootheConfig,
                **kwargs: Any,
            ) -> SubAgent | CompiledSubAgent:
                # Build and return subagent
                ...
    """

    @property
    def manifest(self) -> "SubagentManifest":
        """Return the subagent manifest metadata.

        Returns:
            Manifest with name, version, dependencies, capabilities.
        """
        ...

    async def initialize(self, config: "SootheConfig") -> None:
        """Initialize the subagent module.

        Called once during Soothe startup after dependency resolution.
        Use this to set up resources, validate configuration, establish
        connections, or perform any one-time setup.

        Args:
            config: Resolved Soothe configuration.

        Raises:
            InitializationError: If initialization fails (disables subagent).
        """
        ...

    async def shutdown(self) -> None:
        """Clean up subagent resources.

        Called once during Soothe shutdown. Clean up connections,
        flush buffers, release resources.
        """
        ...

    async def health_check(self) -> "SubagentHealth":
        """Check subagent health status.

        Returns:
            Health status with optional diagnostics.

        Periodic calls allow monitoring of subagent health.
        """
        ...

    def create_subagent(
        self,
        model: str | BaseChatModel | None = None,
        *,
        config: "SootheConfig",
        **kwargs: Any,
    ) -> "SubAgent | CompiledSubAgent":
        """Create a subagent instance.

        This is the main factory method, equivalent to existing
        create_<name>_subagent() functions. Called by resolve_subagents()
        when the subagent is enabled in configuration.

        Args:
            model: LLM model string (e.g., "openai:gpt-4o-mini") or
                   BaseChatModel instance. If None, use subagent default.
            config: Resolved Soothe configuration for dependency resolution.
            **kwargs: Subagent-specific configuration from config.yml.

        Returns:
            SubAgent or CompiledSubAgent compatible with deepagents.

        Raises:
            SubagentCreationError: If subagent creation fails.
        """
        ...
```

### 2. Manifest Schema

Complete metadata declaration for a subagent extension.

```python
from datetime import datetime
from typing import Literal, Any
from pydantic import BaseModel, Field, ConfigDict


class SubagentDependency(BaseModel):
    """A single dependency requirement.

    Attributes:
        type: Dependency category (protocol, tool, library, config, python).
        name: Dependency identifier (e.g., "context", "browser", "langchain").
        version_constraint: PEP 440 version specifier (e.g., ">=0.1.0", "~=2.1").
        required: Whether this dependency is mandatory for subagent operation.
        fallback: Behavior if dependency is missing (disable, warn, fail).
    """

    type: Literal["protocol", "tool", "library", "config", "python"]
    name: str
    version_constraint: str = "*"
    required: bool = True
    fallback: Literal["disable", "warn", "fail"] = "disable"


class SubagentCapability(BaseModel):
    """A capability provided by the subagent.

    Attributes:
        name: Capability identifier (e.g., "web_browsing", "code_generation").
        description: Human-readable capability description.
        input_schema: JSON Schema for expected inputs (optional).
        output_schema: JSON Schema for outputs (optional).
    """

    name: str
    description: str
    input_schema: dict[str, Any] = Field(default_factory=dict)
    output_schema: dict[str, Any] = Field(default_factory=dict)


class SubagentConfigSchema(BaseModel):
    """JSON Schema extension for subagent configuration.

    Attributes:
        json_schema: JSON Schema for subagent-specific config fields.
        defaults: Default configuration values.
        environment: Environment variable mappings (config_key -> ENV_VAR).
    """

    json_schema: dict[str, Any] = Field(default_factory=dict)
    defaults: dict[str, Any] = Field(default_factory=dict)
    environment: dict[str, str] = Field(default_factory=dict)


class SubagentHealth(BaseModel):
    """Health check result for a subagent.

    Attributes:
        status: Health status (healthy, degraded, unhealthy).
        message: Optional diagnostic message.
        details: Additional health metrics or diagnostics.
    """

    status: Literal["healthy", "degraded", "unhealthy"]
    message: str = ""
    details: dict[str, Any] = Field(default_factory=dict)


class SubagentManifest(BaseModel):
    """Complete subagent extension manifest.

    This is the single source of truth for subagent metadata,
    dependencies, capabilities, and configuration requirements.

    Attributes:
        name: Unique subagent identifier (lowercase, hyphenated).
        version: Semantic version string (e.g., "1.0.0").
        description: Human-readable description for the task tool.
        author: Author name or organization.
        homepage: Project homepage URL.
        repository: Source repository URL.
        license: License identifier (SPDX format, e.g., "MIT", "Apache-2.0").
        capabilities: List of capabilities this subagent provides.
        tags: Searchable tags for discovery.
        dependencies: Required and optional dependencies.
        python_version: Python version constraint (PEP 440).
        soothe_version: Soothe version constraint (PEP 440).
        config_schema: JSON Schema for subagent configuration.
        permissions: Requested permission categories.
        trust_level: Required trust level for operation.
        created_at: Manifest creation timestamp.
        updated_at: Last update timestamp.
        keywords: Search keywords.
    """

    model_config = ConfigDict(extra="allow")

    # Core metadata
    name: str
    version: str
    description: str
    author: str = ""
    homepage: str = ""
    repository: str = ""
    license: str = "MIT"

    # Capability declaration
    capabilities: list[SubagentCapability] = Field(default_factory=list)
    tags: list[str] = Field(default_factory=list)

    # Dependencies
    dependencies: list[SubagentDependency] = Field(default_factory=list)
    python_version: str = ">=3.11"
    soothe_version: str = ">=0.1.0"

    # Configuration
    config_schema: SubagentConfigSchema = Field(default_factory=SubagentConfigSchema)

    # Security
    permissions: list[str] = Field(default_factory=list)
    trust_level: Literal["built-in", "trusted", "standard", "untrusted"] = "standard"

    # Metadata
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)
    keywords: list[str] = Field(default_factory=list)
```

### 3. Discovery Mechanism

#### 3.1 Python Entry Points

Extension packages can declare subagents via Python entry points.

**Declaration in `pyproject.toml`**:

```toml
[project.entry-points."soothe.subagents"]
my_subagent = "my_package.subagent:MySubagentModule"
another = "another_package.module:AnotherModule"
```

**Discovery implementation**:

```python
import importlib.metadata
from typing import Sequence

def discover_entry_points() -> list[SubagentManifest]:
    """Discover subagents from Python entry points.

    Returns:
        List of manifests from successfully loaded entry points.
    """
    manifests = []
    try:
        entry_points = importlib.metadata.entry_points(group="soothe.subagents")
        for ep in entry_points:
            try:
                module_class = ep.load()
                module = module_class()
                if isinstance(module, SubagentModule):
                    manifests.append(module.manifest)
            except Exception as e:
                logger.warning(
                    f"Failed to load entry point '{ep.name}': {e}",
                    exc_info=True,
                )
    except Exception:
        logger.debug("No soothe.subagents entry points found")

    return manifests
```

#### 3.2 Config-Declared Extensions

Users can declare extensions directly in `config.yml`.

**Configuration format**:

```yaml
extensions:
  subagents:
    - name: my_custom_subagent
      enabled: true
      module: "my_package.subagent:MySubagentModule"
      config:
        custom_option: value
        another_option: 42

    - name: another_extension
      enabled: false
      module: "another.module:ExtensionClass"
```

**Discovery implementation**:

```python
from typing import Sequence

def discover_config_declared(config: SootheConfig) -> list[tuple[str, str]]:
    """Discover subagents declared in configuration.

    Args:
        config: Resolved Soothe configuration.

    Returns:
        List of (name, module_path) tuples for enabled extensions.
    """
    extensions = []
    if hasattr(config, "extensions") and hasattr(config.extensions, "subagents"):
        for ext in config.extensions.subagents:
            if ext.enabled and ext.module:
                extensions.append((ext.name, ext.module))
    return extensions
```

#### 3.3 Filesystem Discovery

Extensions can be placed in a standard filesystem location.

**Directory structure**:

```
~/.soothe/extensions/
  my_subagent/
    manifest.yml
    module.py  # Contains MySubagentModule class
  another_extension/
    manifest.json
    submodule/
      __init__.py
```

**Discovery implementation**:

```python
from pathlib import Path
from typing import Sequence

def discover_filesystem(
    base_dir: Path = Path("~/.soothe/extensions")
) -> list[SubagentManifest]:
    """Discover subagents from filesystem directory.

    Args:
        base_dir: Base directory for extension discovery.

    Returns:
        List of manifests from valid extension directories.
    """
    manifests = []
    base = base_dir.expanduser()

    if not base.is_dir():
        return manifests

    for manifest_path in base.glob("*/manifest.yml"):
        try:
            manifest = load_manifest(manifest_path)
            manifests.append(manifest)
        except Exception as e:
            logger.warning(
                f"Failed to load manifest from {manifest_path}: {e}",
                exc_info=True,
            )

    # Also check for manifest.json
    for manifest_path in base.glob("*/manifest.json"):
        try:
            manifest = load_manifest(manifest_path)
            manifests.append(manifest)
        except Exception as e:
            logger.warning(
                f"Failed to load manifest from {manifest_path}: {e}",
                exc_info=True,
            )

    return manifests
```

#### 3.4 Priority and Conflict Resolution

When multiple sources provide the same subagent name, priority determines which is used.

```python
from typing import Literal
from dataclasses import dataclass

@dataclass
class RegistryEntry:
    """Entry in the extension registry."""
    manifest: SubagentManifest
    source: Literal["built-in", "entry_point", "config", "filesystem"]
    priority: int
    module: SubagentModule | None = None


class ExtensionRegistry:
    """Registry for discovered extensions with priority handling."""

    # Priority levels (higher = preferred)
    PRIORITY_BUILTIN = 100
    PRIORITY_ENTRY_POINT = 50
    PRIORITY_CONFIG = 30
    PRIORITY_FILESYSTEM = 10

    def __init__(self):
        self._manifests: dict[str, RegistryEntry] = {}

    def register(
        self,
        manifest: SubagentManifest,
        source: Literal["built-in", "entry_point", "config", "filesystem"],
        priority: int | None = None,
    ) -> None:
        """Register a manifest with source and priority.

        Args:
            manifest: Subagent manifest.
            source: Discovery source.
            priority: Priority for conflict resolution (auto-assigned if None).
        """
        if priority is None:
            priority = {
                "built-in": self.PRIORITY_BUILTIN,
                "entry_point": self.PRIORITY_ENTRY_POINT,
                "config": self.PRIORITY_CONFIG,
                "filesystem": self.PRIORITY_FILESYSTEM,
            }[source]

        existing = self._manifests.get(manifest.name)
        if existing and existing.priority >= priority:
            logger.warning(
                f"Ignoring duplicate subagent '{manifest.name}' from {source} "
                f"(existing has priority {existing.priority} >= {priority})"
            )
            return

        self._manifests[manifest.name] = RegistryEntry(
            manifest=manifest,
            source=source,
            priority=priority,
        )
        logger.info(
            f"Registered subagent '{manifest.name}' v{manifest.version} "
            f"from {source} (priority={priority})"
        )

    def get(self, name: str) -> RegistryEntry | None:
        """Get registry entry by name."""
        return self._manifests.get(name)

    def list_all(self) -> list[RegistryEntry]:
        """List all registered entries."""
        return list(self._manifests.values())
```

### 4. Dependency Declaration

#### 4.1 Protocol Dependencies

Subagents can declare which Soothe protocols they require.

**Example**:

```yaml
dependencies:
  - type: protocol
    name: context
    version_constraint: ">=0.1.0"
    required: true
    fallback: disable

  - type: protocol
    name: memory
    version_constraint: ">=0.1.0"
    required: false
    fallback: warn
```

**Resolution**:

```python
def resolve_protocol_dependency(
    dep: SubagentDependency,
    config: SootheConfig,
) -> bool:
    """Check if a protocol dependency is satisfied.

    Args:
        dep: Protocol dependency declaration.
        config: Resolved Soothe configuration.

    Returns:
        True if protocol is available and meets version constraint.
    """
    # Map protocol names to config fields
    protocol_config_map = {
        "context": config.protocols.context,
        "memory": config.protocols.memory,
        "planner": config.protocols.planner,
        "policy": config.protocols.policy,
        "durability": config.protocols.durability,
        "vector_store": getattr(config.protocols, "vector_store", None),
    }

    protocol_config = protocol_config_map.get(dep.name)
    if not protocol_config:
        return False

    # Check if enabled
    if hasattr(protocol_config, "enabled") and not protocol_config.enabled:
        return False

    # Check backend is not "none"
    if hasattr(protocol_config, "backend"):
        if protocol_config.backend == "none":
            return False

    return True
```

#### 4.2 Tool Dependencies

Subagents can declare which tool groups they need.

**Example**:

```yaml
dependencies:
  - type: tool
    name: browser
    required: true
    fallback: disable

  - type: tool
    name: web_search
    required: false
    fallback: warn
```

**Resolution**:

```python
def resolve_tool_dependency(
    dep: SubagentDependency,
    config: SootheConfig,
) -> bool:
    """Check if a tool dependency is satisfied.

    Args:
        dep: Tool dependency declaration.
        config: Resolved Soothe configuration.

    Returns:
        True if tool group is enabled in configuration.
    """
    if not hasattr(config, "tools"):
        return False

    tool_config = config.tools.get(dep.name)
    if not tool_config:
        return False

    # Check if enabled
    if isinstance(tool_config, dict):
        return tool_config.get("enabled", True)
    elif hasattr(tool_config, "enabled"):
        return tool_config.enabled

    return True
```

#### 4.3 Library Dependencies

Subagents can declare required Python packages.

**Example**:

```yaml
dependencies:
  - type: library
    name: langchain-openai
    version_constraint: ">=0.1.0"
    required: true

  - type: library
    name: browser-use
    version_constraint: "~=0.1.0"
    required: false
    fallback: disable
```

**Resolution**:

```python
import importlib.metadata
from packaging.specifiers import SpecifierSet

def resolve_library_dependency(dep: SubagentDependency) -> bool:
    """Check if a library dependency is satisfied.

    Args:
        dep: Library dependency declaration.

    Returns:
        True if library is installed and meets version constraint.
    """
    try:
        version = importlib.metadata.version(dep.name)

        if dep.version_constraint != "*":
            spec = SpecifierSet(dep.version_constraint)
            return version in spec

        return True
    except importlib.metadata.PackageNotFoundError:
        return False
```

#### 4.4 Configuration Dependencies

Subagents can declare required configuration values.

**Example**:

```yaml
dependencies:
  - type: config
    name: providers.openai.api_key
    required: true
    fallback: fail

  - type: config
    name: providers.anthropic.api_key
    required: false
    fallback: warn
```

**Resolution**:

```python
def resolve_config_dependency(
    dep: SubagentDependency,
    config: SootheConfig,
) -> bool:
    """Check if a configuration dependency is satisfied.

    Args:
        dep: Configuration dependency declaration.
        config: Resolved Soothe configuration.

    Returns:
        True if config path exists and is non-empty.
    """
    # Navigate dot-separated path
    parts = dep.name.split(".")
    obj = config

    for part in parts:
        if hasattr(obj, part):
            obj = getattr(obj, part)
        elif isinstance(obj, dict) and part in obj:
            obj = obj[part]
        else:
            return False

    # Check non-empty
    if obj is None:
        return False
    if isinstance(obj, str) and not obj:
        return False

    return True
```

### 5. Configuration Integration

#### 5.1 Configuration Schema Extension

Extend `SootheConfig` to support extension-specific configuration.

```python
from pydantic import BaseModel, Field

class ExtensionSubagentConfig(BaseModel):
    """Configuration for a single extension subagent."""

    enabled: bool = True
    module: str | None = None  # Python import path
    config: dict[str, Any] = Field(default_factory=dict)


class ExtensionsConfig(BaseModel):
    """Configuration for all extensions."""

    subagents: list[ExtensionSubagentConfig] = Field(default_factory=list)


class SootheConfig(BaseSettings):
    # ... existing fields ...

    # Extension system
    extensions: ExtensionsConfig = Field(default_factory=ExtensionsConfig)

    # Dynamic subagent configuration
```

```python
    return errors
```

#### 7.4 Permission Enforcement

```python
from pathlib import Path
from urllib.parse import urlparse


class PermissionEnforcer:
    """Enforce permissions for subagent actions."""

    def __init__(self, permissions: SubagentPermission):
        self.permissions = permissions

    def check_filesystem_access(self, path: Path, mode: str) -> bool:
        """Check if filesystem access is allowed.

        Args:
            path: File path to access.
            mode: Access mode ("r", "w", "x").

        Returns:
            True if access is permitted.
        """
        if self.permissions.filesystem == "none":
            return False

        if self.permissions.filesystem == "read" and mode in ("r", "rb"):
            return True

        if self.permissions.filesystem in ("write", "full"):
            return True

        return False

    def check_network_access(self, url: str) -> bool:
        """Check if network access is allowed.

        Args:
            url: URL to access.

        Returns:
            True if access is permitted.
        """
        if self.permissions.network == "none":
            return False

        if self.permissions.network == "localhost":
            parsed = urlparse(url)
            return parsed.hostname in ("localhost", "127.0.0.1", "::1")

        return True

    def check_subprocess(self) -> bool:
        """Check if subprocess execution is allowed."""
        return self.permissions.subprocess

    def check_env_var(self, name: str) -> bool:
        """Check if environment variable access is allowed.

        Args:
            name: Environment variable name.

        Returns:
            True if access is permitted.
        """
        for pattern in self.permissions.env_access:
            if pattern == "*":
                return True
            if pattern.endswith("*"):
                if name.startswith(pattern[:-1]):
                    return True
            elif name == pattern:
                return True

        return False

    def check_tool(self, tool_name: str) -> bool:
        """Check if tool access is allowed.

        Args:
            tool_name: Tool or tool group name.

        Returns:
            True if access is permitted.
        """
        if "*" in self.permissions.tools:
            return True

        return tool_name in self.permissions.tools
```

```python
    def __init__(self, registry: ExtensionRegistry):
        self.registry = registry
        self.loaded_modules: dict[str, SubagentModule] = {}

    async def load_all(self, config: SootheConfig) -> dict[str, SubagentModule]:
        """Load all discovered and validated extensions.

        Args:
            config: Resolved Soothe configuration.

        await self._discover_all()
        logger.info(
            f"Discovered {len(self.registry.list_all())} subagent extensions"
        )

        validated = self._validate_all()
        logger.info(f"Validated {len(validated)} subagent extensions")

        resolved = self._resolve_dependencies_all(validated, config)
        logger.info(
            f"Resolved dependencies for {len(resolved)} subagent extensions"
```

```python
from typing import Optional


async def load_extension_safe(
    module: SubagentModule,
    config: SootheConfig,

    """Load extension with comprehensive error handling.

    Args:
        module: Subagent module to load.
```

## Security Considerations

### Threat Model

1. **Malicious extensions**: Extensions could contain malicious code
2. **Dependency confusion**: Attacker publishes malicious package with same name
3. **Permission escalation**: Extension requests more permissions than needed
4. **Resource exhaustion**: Poorly written extensions consume excessive resources
5. **Data exfiltration**: Extensions leak sensitive data through network/file access

### Mitigations

1. **Trust levels**: Built-in extensions are trusted by default, third-party require explicit approval
2. **Permission enforcement**: Strict permission boundaries prevent unauthorized access
3. **Dependency verification**: Check library versions against constraints
4. **Resource limits**: Timeout initialization, limit memory/CPU usage
5. **Audit logging**: Log all extension actions for security review
6. **Sandbox consideration**: Future support for containerized/WASM extensions

### Best Practices

1. **Principle of least privilege**: Extensions request minimum permissions needed
2. **Explicit user consent**: User approves elevated permissions
3. **Code review**: Review extension code before installation
4. **Signed extensions**: Future support for cryptographic signatures
5. **Isolation**: Run untrusted extensions with restricted permissions

## Future Enhancements

1. **Cryptographic signing**: GPG/SSH key signing for extensions
2. **Extension marketplace**: Central registry for third-party extensions
3. **Hot reloading**: Load/unload extensions without orchestrator restart
4. **Extension versioning**: Support multiple versions simultaneously
5. **Sandboxing**: Use containers/WASM for untrusted extensions
6. **Extension dependencies**: Extensions can depend on other extensions
7. **Capability negotiation**: Dynamic capability discovery and matching
8. **Health monitoring**: Dashboard for extension health status
9. **Performance metrics**: Track extension performance and resource usage
10. **Rollback support**: Revert to previous extension version on failure
